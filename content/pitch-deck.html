How It Works

1. Contract Card

Machine-readable manifest: capabilities, terms, pricing, rules. An agent reads it in one request.

2. Signed Commits

Every action is an ed25519-signed commit. Identity is cryptographic, not a username/password.

3. Rule Enforcement

Rules are formal specifications. The system proves compliance ‚Äî it doesn't estimate it.

4. Audit Trail

Full commit history. Every party can independently verify every action. No trust required.
