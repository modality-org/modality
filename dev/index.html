Modality

A verification language for AI agent cooperation

The Problem

In a world of AI agents making deals, "trust me" isn't good enough. Agents need a way to negotiate cooperation and prove their commitments — not just promise them.

The Solution

Modality lets agents define cooperation protocols as state machines with temporal modal formulas that constrain behavior. Each party adds their protective rules, and the system verifies that all commitments can be satisfied.

model escrow {
  states { pending, funded, delivered, released, refunded }
  initial pending
  
  transition DEPOSIT: pending -> funded
    +signed_by(/parties/buyer.id)
  
  transition DELIVER: funded -> delivered
    +signed_by(/parties/seller.id)
  
  transition RELEASE: delivered -> released
    +signed_by(/parties/buyer.id)
}

Key Features

Modal Mu-Calculus

Express temporal properties like "eventually paid" or "always authorized" with formal precision.

Cryptographic Predicates

Ed25519 signatures, threshold multisig, oracle attestations — all verifiable on-chain.

Append-Only Contracts

Contracts grow through signed commits. Rules accumulate. The potential space shrinks to mutual agreement.

Model Witnesses

Every rule requires an explicit state machine demonstrating satisfiability. No empty promises.

Quick Start

# Install
git clone https://github.com/modality-org/modality.git
cd modality/rust && cargo build --release

# Create a contract
mkdir my-contract && cd my-contract
modal contract create

# Create identities
modal id create --path alice.passfile
modal id create --path bob.passfile

# Initialize and commit
modal c checkout
modal c set /parties/alice.id $(modal id get --path ./alice.passfile)
modal c commit --all --sign alice.passfile -m "Initial setup"
